    Create `PINOverlay(ModalView)` class following 01-RESEARCH.md Pattern 3 exactly:

    **pin_overlay.py:**
    - Properties: `username = StringProperty("")`, `pin_dots = StringProperty("")`, `error_msg = StringProperty("")`, `mode = StringProperty("login")` (values: "login", "switch", "unlock")
    - `auth_manager = ObjectProperty(None)` — injected from app
    - `on_success = ObjectProperty(None, allownone=True)` — callback(username, role)
    - `_pin_digits = ""` (plain Python str, NOT a Kivy property — per research anti-pattern advice)
    - `on_kv_post(*_)`: init `_pin_digits = ""`
    - `press_digit(digit)`: Append if len < 6, update pin_dots to "bullet * len", clear error_msg
    - `press_backspace()`: Remove last digit, update pin_dots
    - `press_enter()`: Call `auth_manager.validate_pin(username, _pin_digits)`. On success: dismiss + call on_success(username, role). On failure: call `_show_error()`.
    - `_show_error()`: Set error_msg="Invalid PIN", clear digits and dots, shake the pin_card widget (Animation chain per research), schedule error_msg clear after 2 seconds.
    - `show_user_list()`: Build a simple popup or scrollview within the overlay listing `auth_manager.user_names`. Tapping a name sets `self.username = name` and clears PIN. Implementation: Use a `DropDown` or swap the card content to a list of buttons, each setting username and swapping back to numpad view.
    - `open_for_login(auth_manager, on_success)`: Set mode="login", set username=auth_manager.last_user, clear digits/dots, store auth_manager and callback, call `self.open()`.
    - `open_for_unlock(auth_manager, on_success)`: Set mode="unlock", clear digits/dots, set username="" (any Setup/Admin user can unlock), store callback, call `self.open()`. In unlock mode, show user list immediately or filter to Setup/Admin users only.
    - `open_for_switch(auth_manager, on_success)`: Set mode="switch", show user list, store callback, call `self.open()`.

    **pin_overlay.kv:**
    Per research Pattern 3 KV rule:
    - `auto_dismiss: False`, `size_hint: 1, 1`
    - `background_color: 0, 0, 0, 0` (transparent card bg)
    - `overlay_color: 0.031, 0.047, 0.071, 0.95` (near-opaque dark cover)
    - Centered card (AnchorLayout > BoxLayout id:pin_card, size_hint: None,None, width 360dp):
      - Top: Label showing username (or "Select User" if blank), font_size 20sp
      - PIN dots row: Label showing pin_dots, font_size 28sp, letter_spacing 8
      - Error message: Label showing error_msg, color red [0.88, 0.06, 0.06, 1], height 24dp
      - 3x4 GridLayout numpad (cols:3, spacing 4dp):
        - Row 1: 1, 2, 3
        - Row 2: 4, 5, 6
        - Row 3: 7, 8, 9
        - Row 4: Backspace ("<"), 0, Enter ("OK")
        - All buttons: size_hint_y None, height 64dp, font_size 24sp
        - OK button: accent color background
      - Bottom: "Switch User" Button, height 44dp, `on_release: root.show_user_list()`

    IMPORTANT: The pin_card must NOT use pos_hint (per Pitfall 2 in research). Use AnchorLayout as parent with anchor_x='center', anchor_y='center'. The card itself has no pos_hint — this allows the shake Animation to work on card.x.
