  <behavior>
    RED phase tests (write all before implementation):

    CSV-01 Export:
    - test_export_writes_machine_type: export_profile() first row is ['_machine_type', '<type>']
    - test_export_writes_profile_name: export_profile() writes ['_profile_name', '<name>'] row
    - test_export_writes_export_date: export_profile() writes ['_export_date', '<iso-date>'] row
    - test_export_writes_all_scalars: export_profile({'knfThk': 25.5, 'edgeThk': 0.8}) writes one row per scalar with name and value
    - test_export_writes_array_row: export_profile(arrays={'deltaA': [0.0, 0.5, 1.0]}) writes ['deltaA', '0.0', '0.5', '1.0']
    - test_export_csv_parseable: output file can be round-tripped through csv.reader without error
    - test_export_newline_handling: file opened with newline='' (no double \r\n on Windows) — check file bytes

    CSV-02 Parse + Diff:
    - test_parse_returns_metadata: parse_profile_csv() returns dict with machine_type, profile_name, export_date
    - test_parse_returns_scalars: parse_profile_csv() returns scalars dict {var_name: float_value}
    - test_parse_returns_arrays: parse_profile_csv() returns arrays dict {name: [float_values]}
    - test_parse_skips_empty_rows: empty rows in CSV are silently skipped
    - test_diff_only_changed: compute_diff(csv_scalars, current_scalars) returns only rows where values differ
    - test_diff_numeric_comparison: compute_diff({'knfThk': 25.5}, {'knfThk': 25.5}) returns empty (not string mismatch)
    - test_diff_includes_arrays: compute_diff also compares array values and reports changed arrays
    - test_diff_returns_name_current_new: each diff row has keys 'name', 'current', 'new'

    CSV-03 Validation:
    - test_machine_type_mismatch_blocked: validate_import() returns error when CSV machine_type != MACHINE_TYPE
    - test_import_validates_scalar_range: validate_import() returns errors for out-of-range values per PARAM_DEFS min/max
    - test_import_validates_scalar_numeric: validate_import() returns error for non-numeric scalar values
    - test_import_valid_returns_no_errors: validate_import() returns empty error list for valid CSV data
    - test_unknown_array_names_skipped: parse ignores array rows with unrecognized names (extra data is harmless per locked decision)
  </behavior>
  <implementation>
    Create `src/dmccodegui/screens/profiles.py` with these module-level items:

    Constants:
    - MACHINE_TYPE = '4-Axes Flat Grind' (placeholder until Phase 6 adds proper machine-type module)
    - KNOWN_ARRAYS = ['deltaA', 'deltaB', 'deltaC', 'deltaD']

    Pure functions (no Kivy, no controller — pure data transforms):

    1. `export_profile(path: Path, profile_name: str, scalars: dict, arrays: dict, machine_type: str = MACHINE_TYPE) -> None`
       - Opens file with `newline='', encoding='utf-8'`
       - Writes 3 metadata rows: _machine_type, _export_date (datetime.now().isoformat(timespec='seconds')), _profile_name
       - Writes scalar rows: one per var as [var_name, value]
       - Writes array rows: [array_name, val1, val2, ...] with `f'{v:.6g}'` formatting
       - Uses csv.writer (not DictWriter — variable row width)

    2. `parse_profile_csv(path: Path) -> dict`
       - Returns {'machine_type': str, 'profile_name': str, 'export_date': str, 'scalars': {var: float}, 'arrays': {name: [float]}}
       - Rows with key starting with '_' are metadata
       - Rows where key is in PARAM_DEFS var set are scalars
       - Rows with >1 value column are arrays
       - Silently skip empty rows and unrecognized single-value rows

    3. `compute_diff(csv_scalars: dict, current_scalars: dict, csv_arrays: dict, current_arrays: dict) -> list[dict]`
       - Returns list of {'name': str, 'current': str, 'new': str} for changed values only
       - Use `abs(csv_val - ctrl_val) < 1e-9` for float equality (avoids spurious diffs)
       - For arrays: compare element-by-element; if any element differs OR length differs, include in diff
       - Array diff rows: name=array_name, current=comma-joined current values, new=comma-joined new values

    4. `validate_import(parsed: dict) -> list[str]`
       - First check: machine_type match (if mismatch, return single error, skip rest)
       - For each scalar: validate numeric, validate min/max from PARAM_DEFS
       - Return list of error strings (empty = valid)
       - Unknown scalar names are silently ignored (extra data is harmless)

    Import PARAM_DEFS from parameters.py: `from dmccodegui.screens.parameters import PARAM_DEFS`

    Tests go in `tests/test_profiles.py` — pure Python, no Kivy initialization needed. Use `tmp_path` fixture for file I/O.
  </implementation>
