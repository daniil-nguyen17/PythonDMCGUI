  <action>
    Extend `profiles.py` (which already has pure functions from Plan 01) to add the `ProfilesScreen` Kivy Screen class and supporting overlay/dialog classes.

    **ProfilesScreen(Screen):**
    - Properties: `controller = ObjectProperty(None, allownone=True)`, `state = ObjectProperty(None, allownone=True)`
    - `on_pre_enter`: subscribe to state changes for cycle_running interlock, call `_update_import_button()`
    - `on_leave`: unsubscribe from state changes
    - `_update_import_button()`: disable import_btn (disabled=True, opacity=0.4) when state.cycle_running is True
    - `_apply_role_mode(setup_unlocked)`: no-op for now since tab visibility handles role gating (Setup/Admin only see this tab)

    **Export flow:**
    - `on_export_press()`: show ExportNamePopup (Popup with TextInput for profile name + Save/Cancel)
    - `_do_export(profile_name)`: validate name not empty; build file path as `get_profiles_dir() / f'{profile_name}.csv'`; if file exists show overwrite confirmation Popup; then call `_run_export(path, profile_name)`
    - `_run_export(path, profile_name)`: use `jobs.submit()` to read all scalars (one `controller.cmd(f'MG {var}')` per PARAM_DEFS var) and arrays (`controller.upload_array_auto(name)` per KNOWN_ARRAYS, wrapped in try/except) in background thread; on completion call `export_profile()` with collected data; show success/error feedback via label text update

    **Import flow:**
    - `on_import_press()`: open FileChooserOverlay (ModalView with FileChooserListView, path=get_profiles_dir(), filters=['*.csv', '*.CSV'])
    - `_on_file_selected(path)`: call `parse_profile_csv(path)`; call `validate_import(parsed)` — if errors, show error Popup listing all errors and return; then read current values from controller in background via `jobs.submit()`; on completion call `compute_diff()` and show DiffDialog
    - DiffDialog: ModalView with ScrollView containing GridLayout(cols=3) rows (Name, Current, New) for each changed value; Apply + Cancel buttons
    - `_apply_import(parsed)`: use `jobs.submit()` to write scalars (`controller.cmd(f'{var}={val}')`) and arrays (`controller.download_array_full(name, values)`) to controller; then `controller.cmd('BV')` to burn NV; show success feedback

    **get_profiles_dir():** Return `Path(__file__).resolve().parent.parent.parent.parent / 'profiles'` (project root / profiles), create with `mkdir(exist_ok=True)`.

    **FileChooserOverlay(ModalView):**
    - `auto_dismiss: False`, `size_hint: 1, 1`
    - Contains FileChooserListView (id: chooser), Cancel button, Open button
    - `on_file_selected` callback property
    - `confirm_selection()`: reads chooser.selection[0], dismisses, calls callback

    **DiffDialog(ModalView):**
    - `auto_dismiss: False`, `size_hint: 0.9, 0.85`
    - Header row: Parameter | Current | New
    - ScrollView with dynamically added rows
    - Apply (green) + Cancel buttons
    - `build_diff_table(changes: list[dict])`: adds Label widgets per row
    - `on_apply` callback property

    **KV file (profiles.kv):**
    - Use dark theme (import theme), CardFrame pattern
    - Two big buttons (Import, Export) centered on screen, 44dp+ height, full width minus padding
    - Import button id: import_btn
    - Import button text changes to "Import (Cycle Running)" when disabled
    - Export button always enabled when screen is visible
    - FileChooserOverlay, DiffDialog rules defined in KV
    - All colors/spacing match existing dark theme patterns

    **Anti-patterns to avoid:**
    - Do NOT call controller methods on main thread — always use jobs.submit()
    - Do NOT use csv.DictWriter (variable row width)
    - Do NOT use os.getcwd() for profiles path — use Path(__file__)
    - Do NOT block import on unrecognized array names (extra data is harmless per locked decision)
