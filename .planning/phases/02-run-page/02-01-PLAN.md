Implement lifecycle methods following ButtonsSwitchesScreen pattern exactly:
- `on_pre_enter`: Read CPM values once (via jobs.submit), then start 10 Hz poll with `Clock.schedule_interval(self._update_clock, 1/10.0)`
- `on_leave`: Cancel poll event
- `_update_clock(dt)`: Guard with `if not self.controller or not self.controller.is_connected()` then call `self._show_disconnected()` and return. Otherwise `jobs.submit(self._do_poll)`.
- `_do_poll()`: Background thread. Read positions via `MG _TPA`, `MG _TPB`, `MG _TPC`, `MG _TPD`. If IS_SERRATION, also read cycle vars (tooth, pass, depth). Read completion percentage. Post results to main thread via `Clock.schedule_once(lambda *_: self._apply_ui(pos, cycle))`.
- `_apply_ui(pos, cycle)`: Set pos_a/b/c/d as formatted integers with comma thousands separator (`f"{int(val):,}"`). Set cycle status string properties. Calculate elapsed time from a `_cycle_start_time` tracked when cycle starts. Calculate ETA: if `pct > 1.0: eta_s = elapsed / pct * (100 - pct)` else `"--:--"`. Format times as `MM:SS`.
- `_show_disconnected()`: Set all pos_* to `"---"`, clear cycle values.
- `_read_cpm_values()`: Background. For each axis A/B/C/D, try `controller.cmd(f"MG cpm{axis}")`, catch exception (axis may not exist on 3-axis machines), store result. Post to main thread setting cpm_a/b/c/d as e.g. `"1000 cts = 1 unit"` or `""` if read failed.

Implement action handlers:
- `on_start_pause_toggle(state)`: If state == 'down' (Start pressed), send `XQ #CYCLE` (or appropriate start command) via jobs.submit, set `self.cycle_running = True`, record `_cycle_start_time = time.time()`. If state == 'normal' (Pause pressed), send `HX` (halt execution) via jobs.submit, set `self.cycle_running = False`.
- `on_go_to_rest()`: Send `XQ #REST` (or appropriate rest command) via jobs.submit. Reset toggle button to 'normal' state.

Import: `from kivy.clock import Clock`, `from kivy.properties import ...`, `from ..utils import jobs`, `import time`
