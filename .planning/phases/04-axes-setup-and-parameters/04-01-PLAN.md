    1. **Properties:** `_selected_axis = StringProperty("A")`, `_current_step_mm = NumericProperty(10.0)`, position properties for current/rest/start per axis (stored as dicts or individual properties).

    2. **Axis selection:** `select_axis(axis: str)` sets `_selected_axis`. KV bindings will update the visible panel.

    3. **CPM management:** `AXIS_CPM_DEFAULTS = {"A": 1200.0, "B": 1200.0, "C": 800.0, "D": 500.0}`. On `on_pre_enter`, attempt `MG cpmA` etc. via background job; fall back to defaults if read fails or returns 0. Cache in `_axis_cpm` dict.

    4. **Jog:** `jog_axis(axis: str, direction: int)` computes `counts = int(direction * step_mm * cpm)`, then submits `controller.cmd(f"PR{axis}={counts}")` followed by `controller.cmd(f"BG{axis}")` via `jobs.submit()`. Guard: skip if `not self.controller or not self.controller.is_connected()`. Guard: skip if cpm <= 0.

    5. **Step toggle:** `set_step(mm: float)` sets `_current_step_mm`. Called from KV ToggleButton on_release. Three options: 10.0, 5.0, 1.0.

    6. **Live polling:** `on_pre_enter` starts `Clock.schedule_interval(self._poll_tick, 1/3.0)` (3 Hz). `on_leave` cancels. `_poll_tick` submits background job reading `MG _TDA`, `MG _TDB`, `MG _TDC`, `MG _TDD` and posts results via `Clock.schedule_once` to update position display properties. Also read `restPtA/B/C/D` and `startPtA/B/C/D` on initial load.

    7. **Teach:** `teach_rest_point()` and `teach_start_point()` each submit a background job that: reads `_TDA/_TDB/_TDC/_TDD`, writes all 4 `restPtX=val` (or `startPtX=val`) in one semicolon-separated command, then sends `BV`. Guard: skip if `self.state and self.state.cycle_running`. Post success back to UI to update displayed rest/start values.

    8. **Quick actions:** `go_to_rest_all()`, `go_to_start_all()`, `home_all()` each submit `controller.cmd("swGoSetup=1")` (or appropriate variable). Use software variable pattern per CONTEXT.md. `go_to_rest_all` sets `swGoRest=1`, `go_to_start_all` sets `swGoStart=1`, `home_all` sets `swHomeAll=1`. These are best-guess names per research open questions — will be adjusted at integration time.

    9. **Loading flag:** `_loading = BooleanProperty(False)` to suppress UI callbacks during programmatic updates.

    Anti-patterns to avoid:
    - Do NOT use `PA` (position absolute) for jog — use `PR` (position relative) per locked decision
    - Do NOT use `download_array()` — taught points are individual scalar DMC variables per locked decision
    - Do NOT run controller.cmd() on main thread — always via jobs.submit()
    - Do NOT forget to cancel poll clock in on_leave
