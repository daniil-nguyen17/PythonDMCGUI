<objective>
Build the ParametersScreen replacing the current placeholder grid. Setup personnel can view all machine parameters organized in grouped cards (Geometry, Feedrates, Calibration, Positions, Safety), edit values with immediate validation feedback (red=invalid, amber=modified), and batch-apply changes to the controller with automatic read-back verification and NV burn. Operators see the same view but cannot edit.

Purpose: This is how setup personnel tune the machine without raw controller commands. Parameters control feedrates, calibration, geometry, positions, and safety limits.
Output: Working ParametersScreen with grouped cards, dirty tracking, validation, batch apply/read, role-based readonly, and unit tests.

    2. **Properties:** `pending_count = NumericProperty(0)`, `_loading = BooleanProperty(False)`.

    3. **Internal state:**
       - `_param_defs: dict` -- keyed by var name, from PARAM_DEFS
       - `_controller_vals: dict` -- `{var_name: float}` last known controller values
       - `_dirty: dict` -- `{var_name: str}` user-edited strings not yet applied
       - `_field_widgets: dict` -- `{var_name: widget_ref}` for border color updates

    4. **Validation** -- `validate_field(var_name: str, text: str) -> str`:
       Returns "valid", "modified", or "error".
       - Non-numeric text: "error"
       - Out of range (min/max from param def): "error"
       - Zero for pitch/ratio/ctsRev params: "error"
       - Negative for feedrate params: "error"
       - Value matches controller value (within 1e-9): "valid"
       - Value differs from controller value: "modified"

    5. **on_field_text_change(var_name, text):**
       - If `_loading`: return (suppress during programmatic updates)
       - Call validate_field; set border color (red/amber/normal) via `_set_field_state(widget, state)`
       - If "modified": add to `_dirty`; if "valid": remove from `_dirty`; if "error": remove from `_dirty`
       - Update `pending_count = len(self._dirty)`

    6. **apply_to_controller():**
       - Guard: if not _dirty or not connected, return
       - Guard: if state.cycle_running, return (safety -- no BV during cycle)
       - Snapshot dirty dict
       - Background job: send `{var}={val}` for each dirty param, then read back ALL params via `MG {var}`, then send `BV`
       - Post back: update `_controller_vals`, clear `_dirty`, reset `pending_count`, reset all field borders

    7. **read_from_controller():**
       - Background job: for each param in PARAM_DEFS, send `MG {var}`, parse float
       - Post back: set `_loading=True`, update all field widgets with new values, update `_controller_vals`, clear `_dirty`, reset `pending_count`, set `_loading=False`

    8. **Role-based readonly** -- `on_pre_enter`:
       - Check `self.state.setup_unlocked`
       - If False (Operator): set all TextInput widgets to `readonly=True`, hide Apply button (opacity=0, disabled=True)
       - If True (Setup/Admin): set `readonly=False`, show Apply button
       - Always call `read_from_controller()` on enter

    9. **Build UI dynamically** -- `build_param_cards()` called from `on_kv_post`:
       - Group PARAM_DEFS by "group" field
       - For each group: create a CardFrame containing a Label header and rows
       - Each row: 4-column BoxLayout (label, DMC code, TextInput, unit label)
       - Store TextInput widget refs in `_field_widgets`
       - Bind TextInput `on_text` to `on_field_text_change`

    **Border color constants:**
    ```python
    BORDER_NORMAL = [0.118, 0.145, 0.188, 1]  # theme border color
    BORDER_AMBER = [0.980, 0.749, 0.043, 0.9]  # modified highlight
    BORDER_RED = [0.900, 0.200, 0.200, 0.9]    # error highlight
    ```

    Anti-patterns to avoid:
    - Do NOT let on_text fire during read_from_controller -- use _loading flag

    - Do NOT send BV while cycle_running (safety guard)
    - Do NOT hardcode colors -- use BORDER_* constants
    - Do NOT duplicate backOff in both Geometry and Safety -- place it in Safety only
