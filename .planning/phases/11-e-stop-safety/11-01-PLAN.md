  <action>
**jobs.py changes:**
1. Add `_urgent_queue: Queue = Queue(maxsize=1)` and `_cancel_event = threading.Event()` to `JobThread.__init__()`.
2. Add `submit_urgent(self, fn, *args, **kwargs)` method:
   - Call `self._cancel_event.set()` to signal in-flight normal job.
   - Drain any stale entry from `_urgent_queue` (try get_nowait, ignore Empty).
   - Put `(fn, args, kwargs)` onto `_urgent_queue`.
3. Modify `_run()` loop: at the TOP of each iteration (before normal queue get), try `_urgent_queue.get_nowait()`. If found, call `self._cancel_event.clear()` then execute the urgent job. `continue` to loop top. If Empty, proceed to normal queue.
4. After normal queue job completes, also check `_urgent_queue` again (in case submit_urgent fired mid-job).
5. Add module-level `submit_urgent(fn, *args, **kwargs)` convenience function calling `get_jobs().submit_urgent(fn, *args, **kwargs)`.
6. Expose `cancel_event` property (read-only) for testing: `@property def cancel_event(self) -> threading.Event: return self._cancel_event`.

**controller.py changes:**
1. Add `self._address: str = ""` to `__init__`.
2. In `connect()`, after successful `GOpen(address)`, store `self._address = address`.
3. Add `reset_handle(self, address: str | None = None) -> bool` method:
   - `addr = address or self._address`
   - If no addr, return False.
   - Try `self._driver.GClose()`, then `self._driver.GOpen(addr)`.
   - On success: `self._connected = True`, return True.
   - On exception: `self._connected = False`, return False.

**tests/test_jobs.py:** New file with the 5 tests described in behavior block. Use `threading.Event` for synchronization in tests. Keep tests fast (<2s each).
