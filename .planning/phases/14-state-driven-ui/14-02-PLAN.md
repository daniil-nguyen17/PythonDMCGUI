<objective>
Wire motion_active gating on Profiles and Parameters screen buttons using real MachineState.

Purpose: Profile import/apply and parameter Apply must be blocked during active motion (GRINDING/HOMING) and when disconnected. Currently profiles.py gates on the old `cycle_running` flag and parameters.py only checks at call time (no live visual gate). This plan rewires both to use authoritative dmc_state via the motion_active pattern.

Output: Import/apply buttons on Profiles and Apply button on Parameters are visually disabled during GRINDING/HOMING/disconnected, and re-enable on IDLE/SETUP.
</objective>

From src/dmccodegui/screens/profiles.py lifecycle:
```python
def on_pre_enter(self, *args) -> None:
    if self.state is not None:
        self._unsubscribe = self.state.subscribe(
            lambda s: Clock.schedule_once(lambda *_: self._update_import_button())
        )
    self._update_import_button()

def on_leave(self, *args) -> None:
    if self._unsubscribe is not None:
        self._unsubscribe()
        self._unsubscribe = None
```
