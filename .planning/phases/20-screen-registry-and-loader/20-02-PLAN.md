**1. Add `_resolve_dotted_path()` helper to main.py** (module-level or method on DMCApp):
```python
import importlib

def _resolve_dotted_path(dotted: str):
    module_path, attr_name = dotted.rsplit(".", 1)
    module = importlib.import_module(module_path)
    return getattr(module, attr_name)
```

**2. Add `_load_machine_screens(self, sm)` method to DMCApp:**
- Get active type from `mc.get_active_type()`. If empty string, return (unconfigured).
- Look up entry in `mc._REGISTRY[mtype]`.
- Resolve and call `load_kv` function from the entry.
- For each key in `entry["screen_classes"]`, resolve the class, instantiate with `name=canonical_name`, inject `controller` and `state`, call `sm.add_widget(screen)`.
- Wrap in try/except: on failure, show error popup with exception message then `App.get_running_app().stop()`.

**3. Update `build()` in DMCApp:**
- REMOVE lines 109-110 (hard-coded `from .screens.flat_grind import load_kv`).

  - Call `_load_machine_screens()` AFTER `root = Factory.RootLayout()` since the ScreenManager `sm = root.ids.sm` is needed. But machine screen KV must load BEFORE Factory.RootLayout().
  - Split into two steps: (1) resolve and call load_kv BEFORE KV_FILES loop (same position as current _load_flat_grind_kv), (2) instantiate and add screens AFTER root is created.
  
  Concrete approach:
  ```python
  # Step 1: Load machine-specific KV files (before base.kv instantiates RootLayout)
  if mc.is_configured():
      entry = mc._REGISTRY[mc.get_active_type()]
      _load_kv_fn = _resolve_dotted_path(entry["load_kv"])
      _load_kv_fn()
  
  for kv in KV_FILES:

  # Step 2: Add machine screens programmatically

      self._add_machine_screens(sm)
  ```
  
  Where `_add_machine_screens(self, sm)` resolves screen classes, instantiates with canonical names, injects controller/state, and calls sm.add_widget.
