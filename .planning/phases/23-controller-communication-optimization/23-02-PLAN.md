**Message classification constants** (module-level):
```python
# Position message prefixes (from DMC program MG output)
_POSITION_PREFIXES = ("IDLING FOR INPUT", "PRE-LI", "RUNNING", "END REACHED")
STATE_PREFIX = "STATE:"
# Axis label pattern for parsing: "A:123.0  B:456.0"
_AXIS_PATTERN = re.compile(r'([A-Za-z]+):(-?[\d.]+)')
```

**Handler types:**
- `_log_handlers: list[callable]` — freeform MG messages (operator log)
- `_state_handlers: list[callable]` — STATE:N messages, fn(state_int)
- `_position_handlers: list[callable]` — position messages, fn(dict) where dict has axis keys + "prefix"

**_classify_line(line: str) -> tuple[str, Any]:**
- If line starts with STATE_PREFIX: return ("state", int)
- If line starts with any _POSITION_PREFIXES: parse axis values with _AXIS_PATTERN regex, return ("position", dict)
- Otherwise: return ("log", line)

**_loop(address, stop_event):**
- Open gclib handle: `f"{address} --direct --subscribe MG --timeout 500"`
- Call `handle.GTimeout(500)` after GOpen (COMM-06)
- Loop: call handle.GMessage(), classify each line, dispatch to appropriate handlers via Clock.schedule_once
- On GMessage timeout/error: continue loop (check stop_event)

**start(address) / stop():**
- Same pattern as existing _start_mg_reader/_stop_mg_reader in run screens
- start() stores bare address, creates thread, clears stop_event
- stop() sets stop_event, joins with 2.0s timeout

**add_log_handler / add_state_handler / add_position_handler:**
- Each returns an unregister callable (lambda that removes from list)

3. CRITICAL: _classify_line must be a static or class method so it can be tested without thread machinery. The dispatch methods (_dispatch_state, _dispatch_position, _dispatch_log) use Clock.schedule_once to deliver to main thread.
