<task type="checkpoint:human-verify" gate="blocking">
  <name>Task 2: Verify installer on Windows</name>
  <files></files>
  <action>
Present the human verification checklist below. The installer has been compiled in Task 1. The user must run it on a real Windows machine and confirm each acceptance criterion.
  </action>
  <what-built>Complete Inno Setup installer for Binh An HMI, compiled from the .iss script by ISCC.exe.</what-built>
  <how-to-verify>
    1. **Install:** Run `dist\BinhAn_HMI_v4.0.0_Setup.exe` — accept the default install path (C:\Program Files\Binh An HMI)
       - Verify: Minimal wizard (no EULA, no readme page)
       - Verify: "Launch at Windows startup" checkbox is visible and UNCHECKED by default
       - Verify: Final page shows "Launch Binh An HMI now" checkbox, CHECKED by default

    2. **Shortcuts (WIN-03):**
       - Verify: Start Menu has "Binh An" folder containing "Binh An HMI" shortcut
       - Verify: Desktop has "Binh An HMI" shortcut
       - Click the Desktop shortcut — the app should launch (Kivy window with "Binh An HMI v4.0.0" title)

    3. **Add/Remove Programs (WIN-04):**
       - Open Windows Settings > Apps > Installed apps
       - Search for "Binh An" — verify "Binh An HMI" appears with publisher "Binh An"

    4. **Firewall Rules:**
       - Open PowerShell (admin) and run:
         `netsh advfirewall firewall show rule name="Binh An HMI - Galil DR"`
         `netsh advfirewall firewall show rule name="Binh An HMI - Galil TCP"`
       - Verify: Both rules exist (UDP 60007 and TCP 23)

    5. **Uninstall (WIN-04):**
       - Uninstall via Windows Settings > Apps > Installed apps > Binh An HMI > Uninstall
       - Verify: C:\Program Files\Binh An HMI\ is removed
       - Verify: Start Menu shortcut is removed
       - Verify: Desktop shortcut is removed
       - Verify: Firewall rules are removed (re-run the netsh show commands — should say "No rules match")
       - Verify: %APPDATA%\BinhAnHMI\ still exists (user data preserved)

    6. **Optional — Auto-start (WIN-06):**
       - Reinstall with the "Launch at Windows startup" checkbox CHECKED
       - Open Registry Editor: HKCU\Software\Microsoft\Windows\CurrentVersion\Run
       - Verify: "Binh An HMI" key exists pointing to the EXE path
       - Uninstall again — verify the Run key is removed
  </how-to-verify>
