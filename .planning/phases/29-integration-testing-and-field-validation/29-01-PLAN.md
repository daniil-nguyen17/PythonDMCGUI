    Create two deployment guides that allow technicians to install the app without developer assistance.

    **deploy/pi/README.md** — Pi Deployment Guide with three sections:

    1. **Method A: USB/SCP Transfer** (simplest for technicians)
       - Prerequisites: Pi running 64-bit Pi OS Bookworm, USB drive or network access
       - Steps: Copy entire project folder to Pi (USB stick or `scp -r`), then run `sudo bash install.sh`
       - install.sh handles: X11 forcing, apt deps, Galil gclib, venv, pip install, desktop shortcut, screen blanking disable, SSH enable
       - install.sh uses rsync exclusions to skip non-runtime files (.md, .planning/, tests/, .xlsx, .dmc)
       - Expected outcome: Desktop shortcut appears, app launches to PIN login screen
       - Note: install.sh is idempotent — safe to re-run after fixes without starting over

    2. **Method B: SD Card Image** (zero-install for identical hardware)
       - When to use: Deploying to multiple Pis with identical hardware
       - Prerequisites: A working Pi set up via Method A or C, Raspberry Pi Imager or `dd`
       - Steps: On the configured Pi, capture the SD card image (Pi Imager "Use custom" or `dd if=/dev/mmcblk0 of=image.img bs=4M`), write image to new SD card, insert and boot
       - First-boot notes: May need to expand filesystem if using `dd` (raspi-config Advanced > Expand Filesystem)
       - Note: SD card image automation tooling is planned for a future release (FUTURE-02)

    3. **Method C: git clone** (requires internet on Pi)
       - Prerequisites: Pi running 64-bit Pi OS Bookworm with internet access, git installed
       - Steps: `git clone <repo-url>`, `cd <repo-name>`, `sudo bash deploy/pi/install.sh`
       - Same outcome as Method A

    Include a "Troubleshooting" section at the bottom with:
    - kivy_matplotlib_widget may compile from source on aarch64 (20-40 min first install)
    - If screeninfo returns wrong preset, override with `{"display_size": "7inch"}` in `~/.binh-an-hmi/settings.json`
    - If app fails to start, check `~/.binh-an-hmi/logs/app.log`
    - install.sh log at `/var/log/binh-an-hmi-install.log`

    **deploy/windows/README.md** — Windows Deployment Guide:

    1. **For Technicians (Installing)**
       - Copy `DMCGrindingGUI_Setup.exe` to the target Windows 11 machine
       - Run the installer (no admin rights required for per-user install)
       - Launch from Start Menu or Desktop shortcut
       - If Windows Defender quarantines gclib.dll: add the install directory to Defender exclusions (Settings > Virus & threat protection > Exclusions)

    2. **For Developers (Building)**
       - Prerequisites: Python 3.11+, PyInstaller (`pip install pyinstaller`), Inno Setup 6 (optional, for installer .exe)
       - Build command: `deploy\windows\build_windows.bat`
       - Output: `dist\BinhAnHMI\` folder (PyInstaller bundle) + `Output\DMCGrindingGUI_Setup.exe` (if Inno Setup installed)
       - ISCC step is non-fatal: bundle works without Inno Setup

    3. **Data Locations**
       - User data: `%APPDATA%\BinhAnHMI\` (users.json, settings.json)
       - Logs: `%APPDATA%\BinhAnHMI\logs\app.log`
       - Data persists across uninstall/reinstall
