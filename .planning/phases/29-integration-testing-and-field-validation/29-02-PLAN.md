    The Pi install.sh script was built in Phase 26. Three deployment methods were defined in CONTEXT.md. deploy/pi/README.md was created in Plan 01.

    **Step 1: Method A — USB/SCP transfer** (PI-06)
    1. Copy the project folder to the Pi via USB stick or `scp -r`
    2. On the Pi, run: `cd <project-folder> && sudo bash deploy/pi/install.sh`
    3. Wait for install to complete (may take 20-40 min on first run if kivy_matplotlib_widget compiles from source)
    4. Check install log: `cat /var/log/binh-an-hmi-install.log`
    5. Reboot the Pi
    6. Launch the app from the desktop shortcut
    7. Expected: App reaches PIN login screen under X11
    8. Check: `~/.binh-an-hmi/logs/app.log` exists

    **Step 2: Method B — SD card image** (PI-06)
    1. From the working Pi (Method A), capture an SD card image using Pi Imager or `dd`
    2. Write the image to a fresh SD card
    3. Insert into a second Pi (or the same Pi after wiping)
    4. Boot and verify: App desktop shortcut works, reaches PIN login screen

    **Step 3: Method C — git clone** (PI-06)
    1. On a fresh Pi with internet access: `git clone <repo-url>`
    2. `cd <repo-name> && sudo bash deploy/pi/install.sh`
    3. Reboot, launch from desktop shortcut
    4. Expected: Same outcome as Method A

    **Step 4: Display preset validation** (Phase 27 tuning)
    1. For each available touchscreen size (7", 10", 15"):

    4. Verify: DR streaming produces live position data on the run screen
