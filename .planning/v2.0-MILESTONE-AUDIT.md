## Integration Issues (Priority Order)

1. **HIGH — ProfilesScreen exit-setup bug (SETP-01, SETP-08):** `profiles.py` `on_leave` fires `hmiSetp=1` instead of `hmiExSt=0`. Controller stays in STATE_SETUP after leaving Profiles. `on_pre_enter` also lacks smart-enter guard.

2. **MEDIUM — Cold-start E-STOP label (UI-02):** StatusBar shows "E-STOP" at cold start before first connection because `program_running` defaults to False. Should show "OFFLINE".

3. **LOW — _fail_count stale (POLL-03):** `ControllerPoller._fail_count` not reset between disconnect/reconnect cycles. Unlikely to cause issues in practice but is a latent defect.
